# 前端/桌面应用工程师招聘方案

## 职位概述

**职位名称：** 前端/桌面应用工程师  
**优先级：** P0  
**项目：** 花影 (AgentOpsDesktop)  
**汇报对象：** 技术负责人  

## 项目背景

花影是一个跨平台桌面应用，目标是将 CLI Agent、编码助手和自动化工作流统一编排成可管理、可追踪、可治理的 AI 团队。核心技术栈包括 Electron/Tauri + TypeScript/React。

## 核心职责

1. **桌面应用架构与开发**
   - 设计并实现跨平台桌面应用架构（Windows/macOS/Linux）
   - 管理应用生命周期、窗口管理、系统托盘、通知等原生功能
   - 实现应用打包、自动更新、崩溃报告等运维能力

2. **UI 组件库实现**
   - 构建可复用的 React 组件库
   - 实现响应式布局和主题系统
   - 优化组件性能和可访问性

3. **与后端 API 集成**
   - 实现与 Paperclip 控制平面的 API 集成
   - 处理实时数据流（WebSocket/SSE）
   - 管理本地状态和缓存策略

4. **性能优化**
   - 优化应用启动速度和内存占用
   - 实现懒加载和代码分割
   - 优化渲染性能和交互响应

## 任职要求

### 必备技能

- **TypeScript/React 精通**
  - 3年以上 TypeScript 开发经验
  - 深入理解 React Hooks、状态管理、性能优化
  - 熟悉 React 生态（Redux/Zustand/Recoil 等）

- **桌面应用开发经验**
  - 2年以上 Electron 或 Tauri 开发经验
  - 理解主进程/渲染进程架构
  - 熟悉原生 API 调用和系统集成

- **性能优化经验**
  - 有桌面应用性能调优经验
  - 熟悉 Chrome DevTools 性能分析
  - 理解内存管理和泄漏检测

### 加分项

- 有 AI/LLM 应用开发经验
- 熟悉 IPC 通信和进程间协调
- 有开源项目贡献经验
- 熟悉 CI/CD 流程（GitHub Actions 等）

## 面试流程

### 第一轮：技术筛选（30分钟）

- TypeScript/React 基础知识
- 桌面应用开发经验讨论
- 项目经验回顾

### 第二轮：编码测试（60分钟）

- 实现一个 React 组件（带状态管理）
- 解决一个桌面应用相关问题（IPC通信/文件系统）
- 代码风格和问题解决思路评估

### 第三轮：系统设计（60分钟）

- 设计一个桌面应用架构方案
- 讨论性能优化策略
- 评估技术决策能力

### 第四轮：文化匹配（30分钟）

- 团队协作方式
- 学习成长意愿
- 项目理解度
