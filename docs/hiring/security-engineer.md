# 安全工程师招聘方案

## 职位概述

**职位名称：** 安全工程师（应用安全方向）
**优先级：** P2
**项目：** 花影 (AgentOpsDesktop)
**汇报对象：** 技术负责人

## 项目背景

花影是一个跨平台桌面应用，目标是将 CLI Agent、编码助手和自动化工作流统一编排成可管理、可追踪、可治理的 AI 团队。应用涉及 Electron 主进程/渲染进程隔离、IPC 安全通信、RBAC 权限控制、Token 管理、本地 SQLite 数据存储等安全敏感模块。当前安全审计已发现 2 个 High、5 个 Medium、4 个 Low 级别问题，需要专业安全工程师持续治理。

## 核心职责

1. **应用安全治理**
   - 主导 OWASP Top 10 合规审查，推动安全发现闭环
   - 维护 Electron 安全基线（contextIsolation、nodeIntegration、IPC 白名单）
   - 审计 IPC handler 层的输入校验和权限边界

2. **威胁建模与安全设计**
   - 对新功能进行威胁建模（STRIDE/LINDDUN）
   - 设计安全架构方案，评审 PR 中的安全敏感变更
   - 制定 Agent 沙箱隔离策略，防止恶意 Agent 逃逸

3. **安全开发生命周期（SDL）**
   - 建立安全编码规范和 Code Review checklist
   - 集成 SAST/DAST/SCA 工具到 CI/CD 流水线
   - 管理依赖漏洞扫描和 CVE 响应流程

4. **渗透测试与红队评估**
   - 定期对桌面应用进行渗透测试
   - 模拟攻击场景（IPC 注入、环境变量注入、信号注入）
   - 输出可执行的修复建议并跟踪闭环

5. **安全监控与应急响应**
   - 设计安全日志采集和告警策略
   - 建立安全事件响应流程
   - 维护安全运行手册（Runbook）

## 任职要求

### 必备技能

- **应用安全经验**
  - 3年以上应用安全或安全工程经验
  - 精通 OWASP Top 10，有实际漏洞发现和修复经验
  - 熟悉 Electron/Tauri 等桌面应用安全模型

- **威胁建模与安全架构**
  - 掌握 STRIDE 等威胁建模方法论
  - 有安全架构设计和评审经验
  - 理解最小权限原则和纵深防御策略

- **安全工具链**
  - 熟悉 SAST 工具（Semgrep、CodeQL 等）
  - 熟悉 SCA 工具（Snyk、Dependabot 等）
  - 有渗透测试经验（Burp Suite、Metasploit 等）

- **开发能力**
  - 能阅读和理解 TypeScript/JavaScript 代码
  - 能编写安全测试用例和自动化脚本
  - 有 CI/CD 安全集成经验

### 加分项

- 有 Electron 安全审计经验
- 熟悉 IPC 通信安全和进程隔离
- 有 AI/Agent 安全领域经验（Prompt injection、Agent 沙箱逃逸）
- 持有 OSCP、CEH、CSSLP 等安全认证
- 有开源安全工具贡献经验

## 面试流程

### 第一轮：安全基础筛选（30分钟）

- OWASP Top 10 知识考察
- Electron 安全模型理解
- 过往安全项目经验讨论

### 第二轮：威胁建模实战（60分钟）

- 给定花影的 IPC 通信架构，进行威胁建模
- 识别攻击面和潜在威胁
- 提出缓解措施和优先级排序

### 第三轮：安全代码审查（60分钟）

- 审查一段包含安全问题的 TypeScript 代码
- 识别漏洞并给出修复建议
- 评估安全编码习惯

### 第四轮：文化匹配与技术视野（30分钟）

- 安全工程理念和方法论
- 团队协作和沟通方式
- 对 AI 安全趋势的理解
