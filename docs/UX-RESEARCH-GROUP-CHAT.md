# UX Research: 群聊模式 (Group Chat) 编排

> 版本：v1.0 | 作者：UX Researcher | 日期：2026-05-29
> 关联 Issue：[CMPAAA-333](/CMPAAA/issues/CMPAAA-333)

## 研究目标

为 AgentOps Desktop 的多 Agent 群聊模式提供用户体验研究，覆盖：
1. 用户旅程映射
2. 交互模式分析
3. 信息架构建议
4. 可用性风险识别
5. 设计建议

### 1.3 关键触点分析

| 触点 | 用户行为 | 系统响应 | 情绪曲线 |

## 2. 交互模式分析

### 2.1 发言者选择策略

基于竞品分析和用户研究，推荐以下发言者选择策略：

| 策略 | 描述 | 适用场景 | 复杂度 |
|------|------|----------|--------|
| **轮流发言** | Agent 按预设顺序依次发言 | 简单讨论、头脑风暴 | 低 |
| **Manager 分配** | Manager Agent 根据上下文选择发言者 | 复杂任务、需要协调 | 中 |
| **话题触发** | 根据对话内容自动匹配专家 Agent | 专业问题、多领域协作 | 高 |
| **人类指定** | 用户手动选择下一个发言者 | 质量控制、教学场景 | 低 |

**推荐实现顺序：**
1. MVP：轮流发言 + 人类指定
2. V1.1：Manager 分配
3. V1.2：话题触发

**关键设计要点：**
- 每条消息显示：Agent 名称、时间戳、消息内容
- 人类消息用不同颜色/样式区分
- Manager 指令用特殊标记显示
- 支持消息搜索和过滤
- 支持导出对话历史（JSON/Markdown）

### 2.3 人类介入机制

| 介入类型 | 触发方式 | 行为 | 用户感知 |
|----------|----------|------|----------|
| **即时发言** | 输入框发送 | 消息插入对话流 | 直接参与 |
| **暂停对话** | 点击暂停按钮 | 所有 Agent 停止发言 | 掌控感 |
| **指定发言者** | 从列表选择 Agent | 被选 Agent 立即发言 | 引导感 |
| **修改方向** | 输入新话题/指令 | Manager 将指令传达给所有 Agent | 重新聚焦 |
| **终止对话** | 点击结束按钮 | 对话结束，生成总结 | 完成感 |

## 4. 可用性风险识别

### 4.1 高风险问题

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **对话失控** | Agent 陷入无限循环或偏离主题 | 设置最大轮次、超时机制、人类介入按钮 |
| **信息过载** | 多 Agent 同时发言导致信息混乱 | 消息队列、发言者指示器、折叠长消息 |
| **上下文丢失** | 对话过长导致 Agent 忘记早期内容 | 摘要机制、关键信息高亮、上下文窗口管理 |
| **延迟感知** | Agent 响应慢导致用户焦虑 | 加载动画、进度指示、预估时间 |

### 4.2 中风险问题

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **角色混淆** | 用户分不清哪个 Agent 在说什么 | 颜色编码、头像、角色标签 |
| **介入时机** | 用户不知道何时介入最合适 | 介入提示、自然暂停点、建议介入时机 |
| **历史回溯** | 难以找到之前的对话内容 | 搜索、书签、关键消息标记 |

## 5. 设计建议

### 5.1 MVP 功能优先级

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 基础群聊对话 | 核心价值，验证概念 |
| P0 | 对话历史查看 | 基本可观测性 |
| P0 | 人类即时发言 | 基本介入能力 |
| P1 | 轮流发言策略 | 简单可控的发言机制 |
| P1 | 暂停/恢复对话 | 基本控制能力 |
| P2 | Manager 分配策略 | 更智能的发言机制 |
| P2 | 对话导出 | 结果保存和分享 |
| P3 | 话题触发策略 | 高级自动化 |

### 5.2 交互细节建议

**消息输入区：**
- 支持多行输入（Shift+Enter 换行）
- 发送按钮 + 快捷键（Enter 发送）
- 介入按钮（显式标记为人类介入）
- 输入时显示"正在输入..."状态

**对话历史：**
- 自动滚动到最新消息
- 点击消息可查看详情/上下文
- 支持按 Agent 过滤消息
- 支持时间线视图

**参与者面板：**
- 实时显示每个 Agent 状态
- 点击 Agent 可查看详细信息
- 支持手动指定下一个发言者
- 显示 Agent 能力/专长标签

### 5.3 错误处理 UX

| 错误场景 | 用户感知 | 系统行为 |
|----------|----------|----------|
| Agent 响应超时 | 焦虑、不确定 | 显示超时提示、建议介入或重试 |
| Agent 崩溃 | 担忧、困惑 | 显示错误状态、自动移除或重启 |
| 对话死锁 | 沮丧 | 检测死锁、提示人类介入 |
| 网络中断 | 焦虑 | 离线模式、自动重连 |

## 6. 竞品参考

### 6.1 类似产品分析

| 产品 | 群聊模式 | 优势 | 不足 |
|------|----------|------|------|
| **ChatGPT Team** | 多人对话 | 简单易用 | 无 Agent 协作 |
| **Claude Projects** | 上下文共享 | 知识管理 | 无实时群聊 |
| **AutoGen** | 多 Agent 对话 | 灵活配置 | 复杂、难上手 |
| **CrewAI** | 角色扮演 | 任务导向 | 学习曲线陡峭 |

### 6.2 设计灵感

- **Slack/Teams**：频道式对话、@提及、线程回复
- **Discord**：语音频道、角色权限、机器人集成
- **Miro**：实时协作、多人同时编辑、评论系统

## 7. 验证计划

### 7.1 可用性测试场景

| 场景 | 任务 | 成功标准 |
|------|------|----------|
| 发起群聊 | 创建一个新的群聊会话 | 2 分钟内完成配置 |
| 监控对话 | 观察 3 个 Agent 讨论并理解进度 | 能准确描述当前状态 |
| 人类介入 | 在对话中插入一条消息 | 消息被正确插入且 Agent 响应 |
| 导出结果 | 导出对话历史为 Markdown | 文件包含完整对话 |

### 7.2 指标

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 任务完成率 | > 90% | 可用性测试 |
| 首次使用时间 | < 5 分钟 | 新用户引导 |
| 介入成功率 | > 95% | 介入后 Agent 正确响应 |
| 用户满意度 | > 4/5 | 问卷调查 |

## 8. 结论与建议

### 8.1 核心发现

1. **用户需要掌控感**：群聊模式下，用户最担心的是失去对对话的控制
2. **信息过载是主要风险**：多 Agent 同时发言会导致信息混乱
3. **介入机制是关键**：用户需要简单直观的方式参与和引导对话

### 8.2 实施建议

1. **MVP 聚焦基础功能**：先实现简单的轮流发言 + 人类介入
2. **渐进式复杂度**：从简单策略开始，逐步增加自动化
3. **持续用户反馈**：每阶段都进行可用性测试
