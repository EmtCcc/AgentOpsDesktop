# MVP Scope: AgentOps Desktop

## 核心命题

**一人 + 多个 AI 工具的混乱操作 → 一人指挥一个 AI 团队的稳定工作流。**

## MVP 目标

让用户在桌面端完成一个**真实的软件开发闭环**：

> 需求 → 任务拆解 → 多 Agent 并行执行 → 结果汇总 → 人工确认交付

不需要迁移项目、不需要重学命令。连接已有的 CLI Agent，开始编排。

---

## 核心用户旅程

1. **连接 Agent** — 用户在应用中配置已有的 CLI Agent（Claude Code、Codex、Gemini CLI 等），指定可执行路径和工作目录。
2. **创建目标与任务** — 用户创建一个目标（Goal），拆解为若干任务（Task），为每个任务分配 Agent 和工作目录。
3. **并行执行与监控** — 多个 Agent 同时在后台执行任务，用户在统一界面查看实时日志、运行状态、阻塞点。
4. **结果汇总与交付** — 任务完成后，系统汇总各 Agent 的产出（代码变更、文件、报告），用户审阅并确认交付。

---

## 关键功能（3 个）

### 1. Agent Runtime 连接

- 配置本地 CLI Agent 的可执行路径和工作目录
- 支持 Claude Code、Codex、Gemini CLI、OpenCode 等主流 CLI Agent
- 验证连接可用性（health check）
- 本地运行，无需云端账户

### 2. 任务看板与分配

- 创建目标（Goal）和子任务（Task）
- 将任务分配给已连接的 Agent
- 任务状态追踪：pending → running → done / failed
- 单 Agent 串行、多 Agent 并行执行

### 3. 实时监控与日志

- 每个 Agent 运行实例的实时 stdout/stderr 日志流
- 运行状态面板：正在执行、已完成、阻塞中、失败
- 任务产出物预览（变更文件列表、diff 概要）

## 明确非目标（MVP 不做）

| 不做 | 原因 |
|------|------|
| **云端同步 / 多设备** | MVP 聚焦本地单机体验，云端是 v2 的事 |
| **用户账户 / 多租户** | 单用户桌面应用，无需认证体系 |
| **预算/成本控制** | 需要 API 级别的 token 计量，MVP 先不做 |
| **审批门禁 / 人工介入流** | 增加交互复杂度，MVP 用「确认交付」替代 |
| **工作流模板 / 可复用技能** | 需要成熟的任务抽象层，留给后续版本 |
| **定时任务 / 长期任务调度** | 需要持久化调度器，MVP 只支持手动触发 |
| **组织架构 / 角色权限** | 单用户场景无需 RBAC |
| **审计日志 / 合规** | 治理能力是差异化特性，但不是 MVP 阻塞项 |
| **Agent 市场 / 技能商店** | 生态功能，远期目标 |
| **移动端 / Web 版** | 桌面优先，跨平台是后续 |

## 验收标准

MVP 交付时，以下场景必须端到端跑通：

1. 用户在 macOS 上启动应用，配置 1 个 Claude Code Agent
2. 创建一个 Goal：「实现一个 TODO API」
3. 拆解为 2 个 Task：「设计 schema」和「实现 CRUD endpoints」
4. 分配给同一个 Agent 串行执行（或两个 Agent 并行）
5. 实时查看每个 Task 的日志输出
6. 两个 Task 完成后，查看产出物汇总
7. 用户点击「确认交付」，任务标记为 done

---

## 技术约束

- **平台**：macOS 优先（Electron 或 Tauri），后续扩展 Windows / Linux
- **Agent 通信**：通过本地子进程（pty/fork）运行 CLI Agent，捕获 stdout/stderr
- **数据存储**：本地 SQLite，无需外部数据库
- **网络**：MVP 不需要后端服务器，纯桌面端

---

## 风险与依赖

| 风险 | 影响 | 缓解 |
|------|------|------|
| CLI Agent 输出格式不统一 | 日志解析困难 | 先支持 raw output，不做结构化解析 |
| Agent 运行时间不可预测 | 任务超时/卡死 | 设置可配置的超时机制 |
| 并行执行资源竞争 | 磁盘/内存压力 | 限制最大并行数（默认 3） |

## 里程碑

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| M1 | Agent Runtime 连接 + 基础 UI 框架 | 能连接并启动一个 CLI Agent |
| M2 | 任务看板 + 分配 | 能创建 Goal/Task 并分配给 Agent |
| M3 | 实时日志 + 监控 | 能查看 Agent 运行日志和状态 |
| M4 | 结果汇总 + 交付确认 | 完整闭环跑通 |
